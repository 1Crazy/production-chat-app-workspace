## ADDED Requirements

### Requirement: 系统必须支持单聊与群聊会话创建
系统 MUST 支持用户创建单聊和群聊会话，并维护会话成员、角色和会话元数据。

#### Scenario: 创建单聊会话
- **WHEN** 用户选择一个允许联系的目标用户发起聊天
- **THEN** 系统创建或复用唯一单聊会话，并向双方返回相同的会话标识

#### Scenario: 创建群聊会话
- **WHEN** 用户选择多个成员并提交群聊创建请求，且成员数量未超过上限
- **THEN** 系统创建群聊会话、记录创建者和成员角色，并向初始成员广播会话创建事件

### Requirement: 系统必须提供可靠的消息发送与确认机制
系统 MUST 为每条消息记录客户端消息标识、服务端消息标识、发送者、消息类型、会话顺序号和消息状态，并保证幂等处理。

#### Scenario: 客户端重试发送同一条消息
- **WHEN** 客户端携带相同 `clientMessageId` 重新发送先前未确认的消息
- **THEN** 系统不得生成重复消息记录，而是返回既有消息结果或最终确认状态

#### Scenario: 服务端确认消息
- **WHEN** 消息成功写入存储并分配会话顺序号
- **THEN** 系统向发送端返回确认结果，并向在线接收端广播该消息事件

### Requirement: 系统必须支持消息历史同步与断线补偿
系统 MUST 支持按会话顺序号增量同步消息历史，并允许客户端在重连后追平缺失消息。

#### Scenario: 客户端断线后重连
- **WHEN** 客户端携带某会话最后已同步的顺序号发起补偿请求
- **THEN** 系统返回该顺序号之后的消息列表及最新同步水位

#### Scenario: 客户端首次打开历史消息
- **WHEN** 用户首次进入某个已有会话
- **THEN** 系统按分页规则返回最近一页消息，并提供继续向前翻页所需的游标

### Requirement: 系统必须支持已读状态与输入中状态
系统 MUST 维护会话级已读游标，并向符合条件的在线成员广播输入中状态和已读状态变更。

#### Scenario: 更新会话已读状态
- **WHEN** 用户阅读到某会话中的最新消息并上报已读游标
- **THEN** 系统更新该用户的会话已读位置，并重新计算该用户在该会话中的未读数量

#### Scenario: 广播输入中状态
- **WHEN** 用户在会话内进入输入状态且连接在线
- **THEN** 系统向其他在线成员广播短时效的输入中事件，并在超时后自动失效
