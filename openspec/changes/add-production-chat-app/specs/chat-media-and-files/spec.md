## ADDED Requirements

### Requirement: 系统必须提供受控的附件上传授权
系统 MUST 在附件上传前验证用户身份、文件类型、文件大小和业务场景，并向合法请求发放受限时效的上传凭证。

#### Scenario: 获取上传凭证成功
- **WHEN** 用户提交符合白名单类型和大小限制的附件上传申请
- **THEN** 系统返回限时上传凭证、目标对象路径和上传完成确认所需的元数据

#### Scenario: 拒绝非法附件
- **WHEN** 用户提交超限文件、非法扩展名或不允许的 MIME 类型
- **THEN** 系统拒绝发放上传凭证，并返回对应的校验失败原因

### Requirement: 系统必须维护附件消息的生命周期
系统 MUST 将附件元数据、处理状态、预览信息和关联消息状态分开建模，并在上传完成前后维护一致状态。

#### Scenario: 附件上传完成后发送消息
- **WHEN** 客户端完成对象存储上传并提交附件确认请求
- **THEN** 系统登记附件元数据，允许该附件被引用为消息内容，并返回可用于渲染的附件摘要

#### Scenario: 附件仍在处理中
- **WHEN** 附件需要缩略图生成、转码或安全扫描且尚未完成
- **THEN** 系统返回处理中状态，客户端必须能够展示“处理中”而不是误判为失败

### Requirement: 系统必须保护附件访问安全
系统 MUST 仅向有权限访问对应会话的用户提供附件读取能力，并对外链访问设置时效与权限控制。

#### Scenario: 合法成员查看附件
- **WHEN** 当前用户属于附件所属会话且附件状态可读
- **THEN** 系统返回受限时效的下载地址或代理访问结果

#### Scenario: 非法用户访问附件
- **WHEN** 非会话成员或已失去访问权限的用户请求附件内容
- **THEN** 系统拒绝访问，不得泄露对象真实存储路径
