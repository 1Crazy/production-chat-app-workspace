# Code Review Checklist

## 目录与模块

- 新代码是否落在既定 feature / module 中？
- 是否新增了没有边界的 `utils`、`common`、`misc` 类型目录？
- 是否出现跨模块直接访问 repository 的情况？

## 文件职责

- 页面、service、controller 是否只有单一职责？
- 文件是否已经超过约定体量，需要拆分？
- 业务规则是否沉淀在正确层级，而不是堆在 UI 或 controller 中？

## 配置与环境

- 新增配置是否同步更新了四套环境模板？
- 是否把本地密钥或真实凭证写进仓库？
- 可观测性相关改动是否同步更新了指标、日志字段、告警规则或排障文档？
- 备份、恢复或回滚相关流程是否有对应运行手册，而不是只留口头约定？

## 可维护性

- 复杂逻辑前是否有必要注释解释“为什么”？
- 是否留下了明显的样板复制代码，可以进一步抽象？
- 测试或静态检查入口是否同步更新？
- 是否新增或恶化了超过约束阈值的大文件、万能 service 或跨模块耦合？

## 测试与发布

- 核心链路是否至少覆盖单元测试、集成测试、接口契约测试或 widget 测试中的对应层级？
- 新增接口是否验证了请求校验、响应结构和异常路径？
- 发布影响面是否补充了预发验收清单、回滚触发条件和恢复验证步骤？
